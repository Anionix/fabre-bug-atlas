# Docs

These docs explain how to adopt the atlas without making a consumer repository
depend on it at runtime.

## Files

- [`adoption.md`](adoption.md): how to copy the atlas into a repository's
  review process.
- [`redaction.md`](redaction.md): how to turn private observations into public
  guidance.
- [`checklists.md`](checklists.md): compact intake, fix, merge, and public
  evidence checklists.

## Rule

If a consumer repository needs a lesson as closeout evidence, copy the
sanitized lesson into that repository through normal review. Do not rely on a
mutable external document as the evidence itself.
