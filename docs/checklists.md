# Checklists

## Review Bug Intake

- [ ] Read the latest review thread.
- [ ] Check whether the thread is current or outdated.
- [ ] Classify the recurrence.
- [ ] Identify the smallest enforcing layer.
- [ ] Decide whether this is generic enough for the atlas.

## Fix Readiness

- [ ] The fix changes the enforcing layer.
- [ ] A regression guard exists or the no-test reason is explicit.
- [ ] Validation is compact and public-safe.
- [ ] The fix comment names what changed without raw logs.

## Merge Readiness

- [ ] Current unresolved review threads are zero.
- [ ] Outdated unresolved threads are understood.
- [ ] Checks are summarized.
- [ ] Two delayed readbacks were performed for high-churn stacks.
- [ ] Issue closeout evidence points to merged changes.

## Public Evidence

- [ ] Body text scanned.
- [ ] Patch text scanned when a clean patch is claimed.
- [ ] No local paths.
- [ ] No raw environment values.
- [ ] No token-shaped values.
- [ ] No private repository links.
